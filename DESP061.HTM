From: 5314::LEICHTERJ
To:   PARMENTER ! SENT TO PARMENTER
Subject: Unusual laws

Then there was the time Ohm's Law became false by act of Congress.  It seems
that there was a need for standards of measurement for electrical quantities.
Based on (unfortunate) technical advice, Congress defined the ohm in terms of
the resistance of a cubic centimeter of mercury (or some such) under specified
conditions; the volt by the output of some standard cell; and the ampere by
the amount of copper plated out in a given setup in a unit of time.  As it
turns out, there were systematic errors in the experiments used as a basis for
the standards which exceeded the error estimates given, and written into the
law.  As a result, when better experiments were done not long after, Ohm's
Law, which relates these three measurements, was suddenly false!
                                                        -- Jerry

   2.7  RSX-11M COMPATIBILITY MODE - AME

   This section describes VAX/VMS Version 3.3 changes to the Applications
   Migration Executive (AME).

   2.7.1  Retraction of Version 3.2 Patch

   VAX/VMS Version 3.2 included a patch for RSXSHR.EXE which modified how
   the  AME  handled  the C-Mode PC when an exception occurred.  Although
   the modified behavior was closer  to  the  equivalent  behavior  under
   RSX-11M,  some  VAX-11  optional software products depend on the AME's
   incompatible behavior.  Therefore, a patch to retract the Version  3.2
   patch is included in Version 3.3.

From: VORTEX::TSC::MOLSON::D_OREILLY

Subject: A DESPERADO offering...

I've got a nifty-keeno-golly-gee-whiz collection of privileged tasks for
RSX11M+ V2.0/2.1 written in my more insane times (descriptions follow below).
If you would like to get a copy of them for yourself, send me mail at
TSC::MOLSON::OREILLY.

ADB -- Program to look up a selected common region, and print a dynamically-
       updated list of all tasks currently mapped to it

PWQ -- Program to print the contents of the partition wait queues for all
       system-controlled partitions, including the reason the task is in
       the queue (run request or checkpointed)

CKP -- Program to print the contents of the system checkpoint files, to include
       the task name, size (in disk blocks), and starting block number in the
       checkpoint file where the task resides

CKQ -- Program to print the entire contents of the clock queue, not just
       tasks, but also time-based requests queued by the executive

CKD -- Program to print a summary listing by file extension for all UIC's on
       a disk or a specific UIC.  This program is not privileged, but does
       direct I/O to the ACP's.

Please note the above tasks are guaranteed to run only on M+ systems (they
*SHOULD* run on M systems, but I'm too chicken to try them out).

Also, I've got a couple more privileged tasks, but they have an obscure bug
or two -- they tend to execute EOS (end-of-system) occasionally...

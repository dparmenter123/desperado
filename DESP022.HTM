From: REX::LEICHTERJ

Subject: Candidate for sickest DCL programming technique

$! This command file demonstrates an unusual DCL programming technique - a
$! method (NOT recommended for general (any?) use) for testing Boolean condi-
$! tions several times.  Thus, suppose <cond> is a condition and <s1>, <s2>,
$! and <s3> are to be evaluated if and only if <cond> is true.  The following
$! code will do this:
$!
$!      $ Bool = "!"
$!      $ IF <cond> THEN Bool = ""
$!      $'Bool' <s1>
$!      $'Bool' <s2>
$!      $'Bool' <s3>
$!
$! Warning:  You can't conditionalize occurences of things like 'f$verify(0)'
$! this way (i.e. "apostrophed" lexical functions with side-effects).
$!                                                      -- Jerry Leichter
$!
$ TrueBool = "!"
$ if (1 .eq. 1) then TrueBool = ""
$ FalseBool = "!"
$ if (1 .eq. 2) then FalseBool = ""
$'TrueBool'     write sys$output "You should see this"
$'FalseBool'    write sys$output "You should NOT see this"
$'TrueBool'     goto foo
$ write sys$output "You should NOT see this"
$foo:
$'FalseBool'    goto NoSuchLabel
$ write sys$output "You should see this"
